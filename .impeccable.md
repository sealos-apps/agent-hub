# Agent Hub Design Context

## Target Audience
- 在 Sealos 工作区内部署、启动和管理 Agent 的开发者、AI Builder、自动化用户
- 主要使用场景是桌面工作区中的中小窗口，而不是全屏营销站点

## Use Cases
1. 快速创建一个 Agent 实例
2. 为 Agent 自动接入 AI Proxy 与模型配置
3. 查看实例状态、资源规格和初始化进度
4. 打开终端、文件、对话进行验证和排障
5. 在 Sealos 产品体系内获得与 DevBox 一致的操作体验

## Brand Personality
- 系统级
- 克制
- 紧凑
- 可信
- 偏基础设施工具感

## Product Positioning
- Agent Hub 是 Sealos 生态下的一个产品模块，不是独立 SaaS
- 视觉与交互应当贴近 DevBox，同一产品体系下切换时不需要重新学习
- 页面重点是“部署和管理 Agent 实例”，不直接暴露底层 DevBox、Service、Ingress 资源视角

## Interface Principles
- 优先保证信息密度和效率，而不是做强品牌化表达
- 列表页采用工作台式表格结构，不做重卡片瀑布流
- 详情页采用 `Header + Sidebar + Workspace` 骨架
- 默认展示最少且最关键的信息，次要信息通过渐进展开呈现
- 不展示与当前任务弱相关的营销式说明或重复状态块

## Quality Bar
- 这是产品内正式可用页面，不是 MVP 草稿
- 重点是密度、一致性、可维护性、状态反馈清晰
- 与 Sealos DevBox 的产品语言保持一致，但不照搬其业务字段
