# 技术设计: Agent Hub DevBox 风格纠偏

## 设计原则

### 1. 容器优先而不是页面优先
- 所有页面都以受限工作区为前提设计。
- 页面主骨架统一收缩到 `Header + min-h-0 main area`，避免高耸纵向内容。

### 2. 列表优先而不是总览优先
- 首页只保留搜索、创建、实例表格、空态和必要消息提示。
- 统计卡片和冗余说明移除。

### 3. 局部组件承担复杂度
- 将 Agent 列表的名称、状态、资源规格、操作分别拆成小组件。
- 详情页概览拆为独立 overview 组件，避免 page 文件继续膨胀。

### 4. 视觉跟随 DevBox 的中性风格
- 使用更中性的 zinc/white/border/shadow 体系。
- Header 高度、按钮高度、圆角、阴影都收敛到 DevBox 同类范围。

## 结构调整

### 列表页
- 保留顶部 Header。
- 移除首页总览卡片。
- 列表改为紧凑 header row + item row 风格。
- 空态改为 dashed 容器风格。

### 创建页
- 保留返回和提交操作，但压缩 Header 说明文字。
- 左侧保留窄摘要栏，仅显示当前模板和即将部署的关键信息。
- 右侧以配置表单为核心，模板选择与配置区统一收敛为紧凑 section。

### 详情页
- Header 收缩为返回、标题、状态和少量主操作。
- 左侧为窄 icon sidebar。
- Overview 改为若干紧凑信息卡片，不再使用大尺寸展示块。
- `deployment` 独立 tab 合并回 overview，减少导航层级。

## 风险与控制
- 风险: 收缩布局后，现有 chat/terminal/files 工作区可用高度可能不足。
- 控制: 保留 `min-h-0` 和卡片内部滚动，避免工作区溢出。

- 风险: 页面拆分继续进行时，已有 hook 依赖页面态可能造成行为变化。
- 控制: 不改协议和控制器逻辑，只改页面布局与渲染入口。

- 风险: 列表组件拆分后样式可能短期不一致。
- 控制: 同步收口 `Button`、`Input`、`StatusBadge`、`EmptyState` 基础样式。
