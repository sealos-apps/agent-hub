# 变更提案: Agent Hub DevBox 风格纠偏

## 背景
上一轮 Agent Hub 前端重构虽然已经把单页拆成了列表页、创建页、详情页，但页面组织方式仍然偏“大页面产品站”，没有真正对齐 Sealos DevBox 的设计思想。用户已经明确指出两个核心问题：

- 当前整体 UI 组件过大，页面块级层级过深。
- Agent Hub 在 Sealos 内很可能只是一个弹窗或受限工作区，不应该按独立大应用去铺陈。

因此，本轮变更不是继续追加视觉元素，而是对现有实现做一次纠偏，回到 DevBox 的产品语义：紧凑、高密度、表格优先、侧栏清晰、局部能力内聚。

## 核心偏差

### 偏差 1: 首页被设计成“总览页”
- 现状中首页仍有总览卡片、说明块和较重的顶部视觉。
- DevBox 首页本质上是 `Header + List / Empty`，不是多卡片 Dashboard。

### 偏差 2: 创建页仍然偏“向导/宣传页”
- 左侧预览、步骤说明、AIProxy 说明卡占比过大。
- DevBox 创建页强调的是配置流程本身，而不是旁白式说明。

### 偏差 3: 详情页信息块过大
- 当前详情页 Overview 中的大卡片、较宽留白和较重标题层级，不符合 DevBox Detail 的紧凑信息分发方式。
- DevBox Detail 倾向于小侧栏、紧凑卡片、行式信息排布。

### 偏差 4: 组件粒度仍然偏粗
- 列表主组件仍承担过多单元渲染职责。
- 需要继续把列表列、详情概览块、创建摘要块拆成小组件。

## 本次目标
1. 将 Agent Hub 收缩为适合 Sealos 内部工作区/弹窗使用的紧凑布局。
2. 列表页回归 `Header + List/Empty` 主结构，不再承担 Dashboard 职责。
3. 创建页改为“表单优先”的压缩布局，弱化预览型侧边栏。
4. 详情页回归“小侧栏 + 紧凑内容面板”的 DevBox 结构。
5. 进一步拆分大组件，降低单文件复杂度，避免继续在页面层面堆大块 section。

## 影响范围
- `web/src/app/pages/agent-hub/*`
- `web/src/components/business/agents/*`
- `web/src/components/ui/*`
- `web/src/index.css`

## 验收标准
- 首页不再出现大面积总览/统计卡片。
- 创建页与详情页标题区明显收缩，适配受限容器。
- 列表、创建、详情三个页面均以紧凑容器布局为主，空态与表格结构接近 DevBox。
- 列表主名称区域只突出“别名”，`agentName` 作为次级小字展示。
- 组件职责继续下沉，列表和详情不再依赖超大单文件渲染全部内容。
