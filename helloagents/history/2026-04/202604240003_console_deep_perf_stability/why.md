# 变更提案: 控制台深层性能与稳定性治理（文件链路 + 终端渲染）

## 需求背景
当前 Agent 控制台虽然已完成一轮性能优化，但在真实高频使用中仍存在深层问题：
- 文件目录展开与文件预览在远端集群场景下仍然明显偏慢。
- 偶发 `context deadline exceeded` 与请求超时，影响连续浏览。
- 终端在持续输出时仍有卡顿感，不够丝滑。
- 控制台初始进入时目录展开逻辑与预期不一致（默认工作目录展开与手动折叠行为冲突）。
- 曾出现 `Maximum update depth exceeded` 等前端状态联动问题，说明存在边界状态机风险。

这些问题不是单点参数可彻底解决，需按链路做系统治理。

## 变更内容
1. 重构文件资源树初始化策略：默认锚定工作目录优先加载，并保留切换到其他目录（含 `/`）能力。
2. 重构文件请求就绪与重连机制：从轮询等待改为事件驱动就绪门控，降低无效等待与状态抖动。
3. 治理后端文件操作 QoS：优化队列/超时策略与错误语义，降低 `deadline exceeded` 触发概率。
4. 进一步优化终端渲染调度：持续输出下保障输入与滚动实时性，同时维持会话连续性。
5. 收敛前端状态联动与安全边界：修复潜在递归更新风险，补齐 `postMessage` 来源校验。

## 影响范围
- **模块:** Agent 控制台页面、文件 WebSocket Hook、终端渲染调度、后端 WS 会话文件操作、桌面消息解析
- **文件:**
  - `web/src/app/pages/agent-hub/AgentConsoleWindowPage.tsx`
  - `web/src/app/pages/agent-hub/hooks/useAgentFiles.ts`
  - `web/src/components/business/terminal/AgentTerminalWorkspace.tsx`
  - `web/src/components/business/terminal/terminalOutputScheduler.ts`
  - `web/src/app/pages/agent-hub/lib/desktopMessages.ts`
  - `backend/internal/ws/session.go`
- **API:** 主要为现有 WS 消息时序与错误语义优化（优先保持协议兼容）
- **数据:** 前端目录/文件缓存生命周期与命中策略调整

## 核心场景

### 需求: 进入控制台默认展开工作目录且可切换其他目录
**模块:** 资源树初始化与展开状态机
进入控制台后应优先聚焦工作目录，避免从 `/` 全量启动；同时用户可自由切换到任意目录进行浏览。

#### 场景: 首次进入控制台
用户首次打开 Agent 控制台。
- 默认工作目录自动展开并可见子节点
- 目录树不强制加载整个 `/`
- 保留导航到 `/` 与其他路径的能力

#### 场景: 用户手动折叠目录
用户点击目录（包括 `/`）进行折叠。
- 折叠状态应被尊重，不被自动逻辑立即反向覆盖
- 不出现“点击 `/` 无法折叠”

### 需求: 文件浏览与文件预览显著提速
**模块:** 文件请求链路（前端 + 后端）
目录点击与文件打开需要减少排队、无效轮询和重复请求。

#### 场景: 连续展开目录并打开文件
用户快速展开多个目录并连续打开文件预览。
- 目录响应时间稳定，错误可分类
- 同路径请求去重，避免重复网络开销
- `context deadline exceeded` 触发率显著下降

### 需求: 终端在持续输出时仍保持丝滑交互
**模块:** 终端输出调度与渲染
终端高压输出时仍需保证输入响应与滚动体验。

#### 场景: 持续 60s 输出 + 用户持续输入
终端持续输出大量内容且用户同时输入命令。
- 输入与输出均可持续进行
- 不出现明显“卡一顿再刷一段”的节奏
- 队列背压策略可控且提示行为一致

### 需求: 前端状态稳定且安全边界清晰
**模块:** 状态管理与消息接入
避免递归更新与消息注入边界问题。

#### 场景: 高频状态切换
终端/文件/服务标签来回切换并伴随输出。
- 不出现 `Maximum update depth exceeded`
- 页面状态可预测，无异常抖动
- `postMessage` 仅接收可信来源

## 风险评估
- **风险:** 调整目录根策略可能影响既有导航习惯。  
  **缓解:** 提供显式“返回 `/`”入口与路径跳转能力，保证能力不回退。
- **风险:** 文件链路 QoS 调整可能改变超时表现与错误文案。  
  **缓解:** 保持错误码可观测并补充回归测试，确保前端兼容。
- **风险:** 终端调度参数调整不当可能导致输出丢失感知增加。  
  **缓解:** 维持可见提示且建立压测验收阈值，按数据回调参数。
- **风险:** 状态机修复涉及多个 effect，可能引入新联动问题。  
  **缓解:** 增加状态迁移测试与关键路径集成回归。
