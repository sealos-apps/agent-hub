# 变更提案: Agent Hub DevBox 全量 UI 对齐重构

## 需求背景
当前 Agent Hub 虽然已经拆出了列表页、模板页、创建页、详情页，但整体仍然停留在“参考了 DevBox 一部分视觉细节”的阶段，没有真正进入 DevBox 的产品工作台语法：

- 列表页仍保留偏 Web SaaS 的信息组织，Header、列表列语义、空态和操作密度与 DevBox 首页不一致。
- 模板页只保留了搜索和卡片网格，没有复用 DevBox 模板中心的固定宽搜索、左侧分类、overview/category/searchResults 三态切换。
- 创建页依然带有说明型文案和较重的表单化表达，缺少 DevBox 的“左侧辅助轨 + 右侧固定宽配置流”骨架。
- 详情页虽然已有 Header + Sidebar + Content，但概览区仍是自定义信息卡组合，没有采用 DevBox 的 `Basic + LiveMonitoring + Network + Release` 操作工作台结构。
- 后端数据虽然已经提供了创建、更新、启动、暂停、详情读取等能力，但前端页面没有把这些能力组织成 DevBox 那种“列表页控制面 + 详情页操作台”的连续工作流。

本次任务不是继续做样式微调，而是基于本地参考源码 `reference/sealos/frontend/providers/devbox/` 的真实实现，对 Agent Hub 做一次页面级、结构级、状态表达级的完全重构。

## 产品分析

### 目标用户与场景
- **用户群体:** 在 Sealos 内同时使用 DevBox 与 Agent Hub 的开发者、AI 应用工程师、自动化维护人员。
- **使用场景:** 浏览 Agent 实例、选择模板、创建实例、查看运行状态、进入终端/文件/对话、调整资源与模型配置。
- **核心痛点:** Agent Hub 与 DevBox 不属于同一产品语言，导致用户在 Sealos 工作区内切换成本高；当前页面也没有把 Agent 的操作路径组织成稳定的工作台。

### 价值主张与成功指标
- **价值主张:** 让 Agent Hub 成为真正与 DevBox 同语法、同节奏、同交互密度的 Sealos 原生工作台，而不是一个“套了相近配色的业务页”。
- **成功指标:**
  - 列表页、模板页、创建页、详情页都采用 DevBox 风格的固定骨架与控件组织方式。
  - 首页不再出现偏营销或偏说明性的松散布局，信息与动作集中到工作流本身。
  - 详情页概览具备接近 DevBox 的基础信息、运行状态、网络接入、版本/操作信息分区。
  - 前端结构能够明确映射到后端 `/api/v1/agents`、`/api/v1/agents/:name`、`run/pause` 等操作链路。

### 人文关怀
- 使用固定宽和稳定骨架减少用户在 Sealos Desktop 小窗中的视觉形变和认知负担。
- 继续保持 AIProxy 密钥自动注入，不在主界面暴露敏感信息。
- 对创建中、未就绪、不可访问等状态统一给出明确语义，避免用户误判为“实例已经可用”。

## 变更内容
1. 深入校验 DevBox 前后端源码，确认列表、模板、创建、详情页及控制 API 的真实结构。
2. 重建 Agent Hub 的共享视觉语法，使之遵循 DevBox 的固定桌面工作台风格。
3. 重构列表页、模板页、创建页、详情页及相关业务组件，使页面骨架、卡片、列表行、操作区与 DevBox 保持同一产品语言。
4. 在详情页引入更贴近 DevBox 的“基础信息 / 运行状态 / 能力与配置 / 资源与接入”分区，同时保持 Agent 对话、终端、文件能力的现有挂载逻辑。
5. 同步更新 helloagents 知识库、变更记录与历史归档，完整收口本次方案。

## 影响范围
- **模块:** `web/src/app/pages/agent-hub/*`、`web/src/components/business/agents/*`、`web/src/components/ui/*`、`web/src/index.css`、`helloagents/*`
- **文件:** 列表页、模板页、创建页、详情页、共享 header/sidebar/overview、Agent 表格与模板卡、基础按钮/输入框/空态样式
- **API:** 复用现有 `/api/v1/agents`、`/api/v1/agents/:agentName`、`/api/v1/agents/:agentName/run`、`/api/v1/agents/:agentName/pause`、AIProxy token ensure
- **数据:** 不新增持久化结构，主要调整前端视图建模和知识库文档

## 核心场景

### 需求: 列表页必须成为 DevBox 式控制面
**模块:** Agent 列表页
用户打开 Agent Hub 首页时，应看到与 DevBox 首页一致的工具头部、列表头、实例行和空态组织，而不是松散的业务卡片页。

#### 场景: 浏览与控制实例
用户进入 `/agents`。
- 顶部 Header 保持稳定的标题、固定宽搜索框、模板入口和创建按钮。
- 中间主体以固定列宽的表头 + 行项目显示实例名称、状态、资源、更新时间和操作。
- 空态与搜索空态采用 DevBox 风格的独立状态容器。

### 需求: 模板页必须具备 DevBox 模板中心结构
**模块:** Agent 模板页
用户选择模板时，应在与 DevBox 模板市场一致的分类浏览和搜索结构中完成选择。

#### 场景: 浏览与筛选模板
用户进入 `/agents/templates`。
- 页面使用返回头部、ghost tabs、固定宽搜索框。
- 公共模板视图采用左侧分类与右侧内容区结构。
- 卡片尺寸、hover 动作、标签样式与 DevBox 模板卡保持一致。

### 需求: 创建页必须是固定骨架工作流
**模块:** Agent 创建页
用户创建 Agent 时，应在固定宽、左辅右主的工作台中完成配置，而不是在长说明表单中滚动。

#### 场景: 使用模板创建实例
用户从模板页进入 `/agents/create`。
- Header 为返回 + 标题 + 操作按钮。
- 左侧显示模板摘要、资源概览和 AIProxy 接入状态。
- 右侧使用固定宽卡片组织运行时、实例名称、资源、模型等配置。

### 需求: 详情页必须接近 DevBox 的操作工作台
**模块:** Agent 详情页
详情页需要成为 Agent 的稳定操作台，而不是只放几个 overview 卡片。

#### 场景: 查看实例详情与切换能力
用户进入 `/agents/:agentName`。
- Header 显示返回、名称、状态、删除/终端/启动暂停等操作。
- 左侧是窄图标 Sidebar。
- Overview 页以基础信息、运行状态、能力、资源、接入等区块组织，并维持 DevBox 的中性卡片语法。

### 需求: 前后端状态语义需要统一
**模块:** 状态模型与控制流
前端页面必须明确承接后端的状态语义和操作接口，避免只做静态皮肤。

#### 场景: 实例从创建中变为可用
用户创建或启动 Agent 后回到列表页或详情页。
- 列表页轮询保持创建态与 ready 态刷新。
- 详情页操作按钮和能力状态根据后端返回字段变化。
- 页面文案明确区分 running / creating / stopped / error 与 ready / bootstrap message。

## 风险评估
- **风险:** 一次性调整页面骨架和共享样式，可能影响已有业务组件布局。
- **缓解:** 通过共享 DevBox 风格类名和逐页重构，避免把样式分散写死在各页面。
- **风险:** Agent 模板数据量少于 DevBox，强行复制全部状态机会产生空结构。
- **缓解:** 复用 DevBox 的结构语法，但按 Agent Hub 当前模板规模做最小可行落地。
- **风险:** 详情页目前没有独立 detail API 拉取链路，部分字段只能使用列表态快照。
- **缓解:** 优先用现有 `AgentListItem` 和后端已返回字段组织工作台；不足处通过文档明确记录。
